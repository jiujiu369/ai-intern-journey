#	三大核心概念极简总结	#
##	Token —— 大模型的最小处理单位
	-	不是汉字、不是单词，是大模型专属的「认字颗粒」。中文1个字≈1.3-1.5个Token，英文1个单词≈1个Token。
	-	核心价值：计费标准 + 长度限制
	-	RAG关联：Day5做文本分块、清洗的根本原因——Embedding模型有最大Token上限，超长文本无法向量化，必须切块。
##	Embedding（词嵌入/向量化）—— 让电脑读懂语义
	-	核心本质：文字 → 数字向量
	-	人类看文字意思，电脑看不懂文字，只认数字空间距离。语义相似的文本，向量在高维空间中距离极近；语义无关/相反，距离极远。
	-	核心铁律：同一套文本、同一个模型，生成向量固定；不同模型向量绝对不能混用。
	-	RAG核心作用：RAG告别传统关键词匹配，靠Embedding实现语义匹配，是智能检索的核心。
##	余弦相似度 —— 语义相似度打分器
	-	专门计算两个高维向量的相似程度，取值范围 [-1,1]
			≈1：语义高度相似（RAG召回的目标文本）
			≈0：语义完全无关
			≈-1：语义完全相反
	-	RAG检索流程：用户问题向量化 → 和所有文档块向量计算余弦相似度 → 排序取Top-k，完成语义召回。
	
#	本次实验架构（3 个解耦代码模块	##
##	embedding_local.py 本地向量化模块
	-	依赖库：sentence-transformers、numpy
	-	模型：BAAI/bge-small-zh-v1.5 轻量中文向量模型输出512维向量
			BAAI/bge-small-zh-v1.5	512 维		约 100MB 内存占用
			BAAI/bge-base-zh-v1.5	768 维		约 400MB 内存占用
			BAAI/bge-large-zh-v1.5	1024 维		1.3GB 左右内存占用
	-	功能：清洗文本、离线生成 768 维语义向量，无需任何外网 / API 密钥
	-	优势：规避公司内网防火墙拦截、线上 API 鉴权失败、额度扣费问题，一次下载永久离线使用
##	cos.py 余弦相似度计算模块
	-	依赖库：numpy
	-	功能：手写实现余弦相似度数学公式，不调用封装好的工具函数，直观看懂向量打分逻辑
	-	输入：两段文本生成的浮点向量列表；输出：0~1 之间的相似度分数
##	test_demo.py 交互测试入口
	-	功能：支持自定义输入三段文本（基准句、相近句、无关句），串联向量生成 + 相似度计算，打印直观对比结果
	-	适配方案：增加系统路径兼容代码，解决同目录文件导入ModuleNotFoundError报错
#	实操实验案例与结果解读	##
-	测试输入文本
		基准文本：今天是晴天
		相近语义文本：今天天气很好
		无关语义文本：今天吃什么
-	实验输出结果
	plaintext
		===== 本地离线Embedding语义相似度实验 =====
			语义相近两句话相似度：0.7482
			语义无关两句话相似度：0.5422
	**无法连接则配置镜像环境：$env:HF_ENDPOINT="https://hf-mirror.com"**
##	结果分析
-	相近句分数 0.7482：两句核心语义都是描述当日天气，向量在高维空间夹角很小，模型识别出同义语义；
-	无关句分数 0.5422：仅共享 “今天” 词汇，核心话题完全无关，语义匹配度显著降低；
-	对比规律：语义重合度越高，余弦分数越接近 1；话题完全割裂，分数明显下降，验证 Embedding + 余弦相似度的语义识别能力。
#	线上 API 踩坑总结（本次尝试过 apikl、DeepSeek 双渠道）	##
-	apikl 中转 GPTPro 分组：仅支持text-embedding-3-small，不支持 bge 系列中文向量模型，填错模型名返回model_not_found；
-	DeepSeek 官方接口：公司内网防火墙拦截出站 HTTPS 连接，无代理直接报APIConnectionError；
-	线上通用痛点：需要密钥、消耗余额、网络环境受限，内网开发场景稳定性差；
-	本地模型优势：无网络依赖、免费、不受防火墙限制，适合学习、内网调试 RAG 底层逻辑。
#	拓展延伸（RAG 完整链路铺垫）	##
-	向量持久化：后续可将文档向量存入向量数据库（Chroma、FAISS），避免每次检索重复生成向量，提升检索速度；
-	阈值筛选：实际业务中设置相似度阈值（如 0.6），低于阈值直接丢弃无关文档，减少大模型输入冗余；
	模型升级：本地可替换bge-large-zh-v1.5高精度向量模型，代价是文件更大、推理速度变慢；线上可切换 OpenAI/DeepSeek 官方向量接口，适配公网可访问环境。
#	关键避坑要点	##
	向量模型和对话模型不可混用：gpt-5.5、deepseek-chat 等对话模型无法调用/embeddings向量接口；
	向量不能跨模型对比：本地 BGE 向量、OpenAI 向量维度不同，直接计算余弦相似度无意义；
	Windows 下载 HuggingFace 模型慢：可配置HF_ENDPOINT=https://hf-mirror.com国内镜像加速；
