# 提示词工程 5 大核心技巧（RAG 必用）
1. 角色设定：锁定模型身份，统一输出风格
2. 指令明确化：杜绝模糊描述、杜绝答非所问
3. Few-shot 少样本：用示例替代文字规则，格式更稳
4. 输出格式约束：标准化，方便代码解析
5. 边界与幻觉约束：RAG 降幻觉核心手段


# 核心实验内容
1. 基础提示词对照实验
对比三层提示词的输出差异：
1. 无约束模糊提问：易幻觉、无来源、格式混乱
2. 角色 + 基础指令：专业性提升，减少胡编乱造
3. 完整工程级约束：可控溯源、固定拒答、零拓展
2. 自研可控 RAG Prompt（本项目核心产出）
# 全链路自主可控设计：
- 内置知识库文档（固定上下文，杜绝脏输入）
- 交互式手动输入问题（支持标准/误导/超范围/自定义）
- 强规则锁死模型行为，模型无自主发挥空间
# 五大硬性防幻觉规则：
1. 仅允许基于参考文档回答
2. 无知识必须固定话术拒答
3. 禁止常识补充、禁止引申、禁止拓展
4. 所有结论必须标注来源
5. 回答简洁、无闲聊、无冗余
