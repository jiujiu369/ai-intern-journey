#	LLM四大核心概念
	Token，Embedding，Attention，Transformer
-	**token**:LLM的工程最小单位
	**三大工程约束**
- 		上下文窗口：4k/8k/128k 指最大 Token 容量，决定单次可输入文本上限
- 		计费规则：输入、输出双向计费，输出更贵
- 		长文本限制：超长文本必须分块、截断、压缩，否则直接报错
	Token 是大模型处理文本的最小单位，也是上下文限制和计费的核心依据。RAG 分块、对话截断、长文本压缩都是为了适配 Token 窗口限制。
-	**Embedding:**把文字变成一串数字向量，语义越相近，向量距离越近
	**工程红线**检索向量和文档向量必须同一个 Embedding 模型，否则完全不匹配
-	**Attention:**解决长文本上下文关联弱、语义跑偏、问答不准的问题。
	RAG 给一堆上下文时，模型靠 Attention 精准聚焦参考片段，给出正确答案。
-	**Transformer:**Transformer 是当前大模型的标准底座，依靠注意力机制实现强大的语义理解和生成能力，可扩展性极强

#	四大概念完整串联逻辑
	Transformer（底座） → 依靠 Attention（聚焦语义） → 读取 Embedding（文本向量） → 所有输入输出受 Token（工程限制）约束

#	实操
1.	Token 计数实验
-	接入deep seek API
-	中文不能按 1:1 估算，必须按 Token 计算；
-	4k 模型实际可用汉字大约 2800～3000 字；
-	RAG 分块、上下文截断、成本预估，全部依赖 Token 统计。

2.	上下文对照实验（RAG 终极原理）
-	无上下文：模型靠训练数据回答，容易不知道、泛化、出错；
-	有正确上下文：精准回答业务问题，无幻觉；
-	有错误上下文：模型会跟随错误内容回答；
-	结论：大模型不存储实时/私有知识，模型答什么，完全取决于你喂什么上下文。这就是 RAG 消除幻觉、适配私有业务的底层逻辑

**RAG 解决大模型幻觉的核心原理**
1.	为什么大模型会产生幻觉
		大模型的所有输出，依赖训练阶段固化的静态知识库，存在两个致命缺陷：
			知识滞后、有盲区：无法学习训练后的新知识、企业私有业务知识（公司制度、内部流程）；
			自由生成不可控：模型为了保证语句通顺，会自动脑补、编造看似合理但虚假的内容，也就是幻觉。
2.	RAG 是如何根治幻觉的？
		RAG 不修改模型、不训练模型，只通过「外部真实知识库约束生成」解决问题，核心三步：
			外部检索真实数据：从私有/最新知识库检索真实、权威、时效性的参考文本；
			强制注入上下文：把检索到的真实资料拼入 Prompt，作为模型唯一答题依据；
			约束模型生成逻辑：强制模型基于参考内容作答，无相关内容就说不知道，杜绝自由编造。
3.	本质核心
		大模型原生是「生成优先」，容易瞎编；
		RAG 把它变成「检索优先」，模型答什么不再靠训练记忆，而是靠实时输入的真实上下文，从源头大幅消除幻觉
